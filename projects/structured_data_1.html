HR Pipeline

Description: Brief description of the HR pipeline project.

Technologies Used:

  • Python: Primary programming language for developing the ETL pipeline.
  • PySpark: For distributed data processing to handle large datasets efficiently.
  • Pandas: For data manipulation and conversion between Spark and Pandas DataFrames.
  • Jupyter Notebook: For data visualization and analysis.
  • Matplotlib: Library for creating visualizations in Python.
  • Seaborn: Data visualization library based on Matplotlib, providing a high-level interface for statistical graphics.
  • Hadoop: Configured to work with Spark for file handling.
  • VS Code: Integrated development environment used for coding and debugging.

Challenges: 1. File Handling and Permissions Challenge: Encountered FileNotFoundError and PermissionError when reading the input file and overwriting the output file. Solution: Ensured the input file existed at the specified path, closed any open instances of the output file, adjusted file permissions using os.chmod, and removed existing files programmatically before writing new data. 2. Data Transformation Challenge: Handling inconsistent date formats, missing values, and normalizing job titles. Solution: Used Spark SQL functions to format dates consistently, calculated employee tenure in days, dropped rows with critical missing values, and used regex to normalize job titles. 3. Environment Configuration Challenge: Issues with setting up the Hadoop home directory and configuring Spark environment variables. Solution: Correctly set the HADOOP_HOME environment variable and configured Spark to use the local file system. Ensured all dependencies were correctly installed and paths were correctly set. 4. Performance Optimization Challenge: Processing large datasets efficiently. Solution: Leveraged PySpark for distributed processing and ensured efficient use of resources by tuning Spark configurations.
